# Glossary

- Spec: A markdown doc defining one feature with acceptance criteria.
- Constitution: Non-negotiable engineering laws for this repo.
- ADR: Architecture Decision Record (why we chose something).
- Evidence: Proof artifacts (screenshots/logs) demonstrating the feature.
- API Boundary: The single place network calls are made from UI code.
- Token Authority: The single place token is stored/cleared/read.
- Owner-only: User can only see/modify their own tasks; others behave like 404.
